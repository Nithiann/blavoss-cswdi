Casus beschrijving - Festival tickets

In deze casus wordt de ticket verkoop van festivals gepland. Voor de casus wordt er een festival gepland. Een gebruiker kan hier de tickets voor kopen. Een gebruiker moet ingelogd zijn om tickets te kunnen kopen. Voor het doeleinde van deze casus zal er een toegewezen account zijn dat festivals kan aanmaken. De andere gebruikers zijn normale users. Voor tussenbeoordeling 1 is alleen de user entiteit met zijn crud-functionaliteiten gemaakt. Er hoeft voor nu dus nog niet ingelogd worden om bij deze functionaliteiten te komen. Om bij de user te komen kan voor nu op de navigatiebalk op users geklikt worden of hier geklikt worden.

In deze casus wordt gebruik gemaakt van een monorepo. De backend wordt gemaakt in nextjs met een frontend in angular. de angular applicatie wordt met behulp van tailwindcss en flowbite in elkaar gezet. De database is gemaakt in mongoDB. er zal later ook een neo4j database gemaakt worden, zodat er festivals aan gebruikers voorgesteld kunnen worden op basis van hun vorige festivals.

Entiteiten

er worden 4 entiteiten gemaakt voor deze casus. de user-entiteit, deze bevat de gegevens van de gebruiker. De artist-entitiet, deze bevat informatie over een artiest. De Festival-entiteit, deze bevat de gegevens van een festival. De TicketSales-entiteit, deze bevat de gegevens van een ticketSales.

User entiteit

  • Id: ObjectId
  • Email: string
  • hash: string
  • firstName: string
  • lastName: string
  • DateOfBirth: Date
  • Gender: enum(string)
  • tickets: [...Ticketsales]

    • Artist entiteit

  • Id: ObjectId
  • name: string
  • genre: enum(string)
  • Description: string
  • Festivals: [...Festivals]

    • Festival entiteit

  • Id: ObjectId
  • name: string
  • Location: string
  • startDate: datetime
  • EndDate: datetime
  • Description: string
  • ticketPrice: number
  • genre: enum(string)
  • Artists: [...Artists]
  • Tickets: [...Tickets]

    • TicketSales entiteit

  • Id: ObjectId
  • UserId: ObjectId
  • FestivalId: ObjectId
  • TicketAmount: number
  • PurchaseDate: datetime
  • PaymentStatus: enum(string)
  • PersonalizationStatus: enum(string)

    • Er is een many-to-many relatie aanwezig tussen artiest en festival. Meerdere artiesten kunnen voor meerdere festivals ingepland worden. De ticketsales entiteit werkt ook als koppeltabel tussen user en festivals.

    voor tussenbeoordeling 2 zijn de artists en festivals entiteiten gerealiseerd. Het is mogelijk om een artiest aan te maken, verwijderen, updaten en natuurlijk lezen. Het is mogelijk om ook een festival aan te maken, verwijderen, updaten en lezen. Je kan een artiest toevoegen aan een festival en je kan ook de fesitvals zien die artiesten hebben staan of gehad hebben. op bijna alle routes staan auth guards of admin guards. Het is om deze reden handig in te loggen om de applicatie te testen. binnen de CI/CD straat wordt de applicatie gebouwd en getest. Deze moet slagen wilt de app gaan deployen.
